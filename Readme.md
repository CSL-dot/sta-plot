# <center>软件说明1.0</center>

&emsp;&emsp;本软件基于Python代码开发，主要用途为GNSS接收机连续打点以及固定点持续观测的指标统计和绘图处理。目前主要处理南方接收机的数据。

## 1.数据介绍

&emsp;&emsp;原始数据内容主要包括：点名，北坐标，东坐标，高程，编码，解状态，RMSH，RMSV，卫星数，PDOP，日期， 时间，里程，偏距，天线高，实际平滑次数，基站空间距，基站平面距，差分Age，存储类型，点名，存储位置，中桩里程，中桩高程，GPS时间。
&emsp;&emsp;数据处理会删去一些不重要的数据。

## 2.成果输出

&emsp;&emsp;统计指标输出：
&emsp;&emsp;1)单文件处理：两种文件都会输出平均RMSV、RMSH、卫星数、PDOP、Diff Age，固定点文件相较于移动点文件会额外输出点的平均坐标以及标准差；
&emsp;&emsp;2)对比文件处理：大体与单文件处理相同，会额外输出对比信息。

&emsp;&emsp;绘图输出：RMSV、RMSH、PDOP、Sat Num、Diff Age的时序变化。

## 3.单文件处理使用方法

&emsp;&emsp;1)导入数据：不限.dat，.csv等文本形式文件，文件不需配置列标题，列标题已写入程序；
&emsp;&emsp;2)选择移动点观测文件还是固定点观测文件，主要影响指标输出，固定点会输出坐标平均值及坐标标准差；
&emsp;&emsp;3)选择处理时段，后续处理都基于选取时段；
&emsp;&emsp;4)时间分辨率为接收机打点时间间隔，默认1s，可根据GNSS数据的时间分辨率修改；
&emsp;&emsp;5)图表有默认配置，也可自定义之后，开始分析并绘图，然后即可导出图表、报告；对导出图不满意，可重置绘图区，调整图表配置，重新导出；
&emsp;&emsp;6)当前文件处理结束，可重置整个文件，进行下一个文件的处理

## 4.多文件处理使用方法

&emsp;&emsp;总体与单文件处理方法类似，区别为需指定参考文件和待处理文件；选择移动点观测文件还是固定点观测文件，主要影响指标输出。
